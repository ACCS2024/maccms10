| Receive.php | 7 |
| Danmaku.php | 7 |
| Live.php | 6 |
| 其余文件 | 合计约 43 |

**替换规则**：

```php
// 模式 1：直接调用方法
model('Vod')->getListByParam($param)
→ (new \app\common\model\Vod())->getListByParam($param)

// 模式 2：赋值后多次使用（提取变量）
$res1 = model('User')->xxx();
$res2 = model('User')->yyy();
→ $user = new \app\common\model\User();
  $res1 = $user->xxx();
  $res2 = $user->yyy();

// 模式 3：在条件或链式中
if (model('Order')->save($data)) { ... }
→ if ((new \app\common\model\Order())->save($data)) { ... }
```

**批量替换辅助**（处理最简单的单行模式）：
```bash
# 先用 sed 处理简单模式：model('Xxx') 整词替换为 new \app\common\model\Xxx()
# 注意：sed 无法处理驼峰大小写映射，建议用 PHP 脚本或手动处理
# 验证替换结果：
grep -rn "model(" application/api/controller/ | grep -v "//\|#"
# 期望：0 行
```

**注意**：替换完成后删除 `model()` shim 不在本阶段，P5-01 才移除。

## P2-03 API 控制器 input() 替换

**目标**：api/controller/ 下约 5 处 `input()` 调用

**先确认位置**：
```bash
grep -rn "input(" application/api/controller/ --include="*.php" | grep -v "//\|#\|htmlspecialchars_decode\|stripslashes"
```

**替换规则**：
```php
input('post.')       → $request->post()
input('post.id')     → $request->post('id')
input('get.page', 1) → $request->get('page', 1)
input('param.xxx')   → $request->param('xxx')
input('param.')      → $request->param()
```

TP8 中 `$request` 可通过方法注入获取：
```php
// 方法签名中注入
public function index(\think\Request $request) { ... }
// 或通过 facade
\think\facade\Request::post('id')
```

## P2-04 创建 route/api.php

**目标文件**：`route/api.php`（新建，同时创建 `route/` 目录）

**来源**：`application/route.php`，提取 API 相关路由段

**先查看现有路由**：
```bash
cat application/route.php
```

**TP8 路由文件格式**：
```php
<?php
use think\facade\Route;

// 从 application/route.php 中提取 API 相关路由，转为 TP8 格式
// TP5 数组格式：'api/vod/index' => ['api/vod/index', [], []]
// TP8 链式格式：Route::get('api/vod/index', 'api/vod/index');

// API 路由通常是这种形式：
Route::group('', function() {
    // 提取 route.php 中 ENTRANCE=api 段的路由
});
```

**TP5 → TP8 路由转换对照**：

| TP5 | TP8 |
|-----|-----|
| `'pattern' => ['id' => '\d+']` | `Route::pattern(['id' => '\d+'])` |
| `'path' => ['controller/action', [], []]` | `Route::rule('path', 'controller/action')` |
| GET 方法限制 | `Route::get(...)` |
| POST 方法限制 | `Route::post(...)` |
| 无限制 | `Route::any(...)` 或 `Route::rule(...)` |

## P2-05 API 接口回归测试

**目标**：确认 API 模块关键接口响应正确，字段与 master 分支对比无差异。

**测试接口清单**：

```bash
BASE="http://your-host/api.php"

# 视频列表
curl -s "$BASE/vod/index" | python3 -m json.tool | head -30

# 视频详情
curl -s "$BASE/vod/detail?vod_id=1" | python3 -m json.tool | head -30

# 搜索
curl -s "$BASE/vod/search?wd=test" | python3 -m json.tool | head -20

# 用户注册（验证 Model + DB 写入）
curl -s -X POST "$BASE/user/reg" -d "username=test_tp8&password=123456&email=tp8@test.com"
```

**通过标准**：
- 响应格式为有效 JSON
- `code` 字段存在（1=成功，0=失败）
- 数据字段（list/info 等）结构与 master 分支一致
- 无 PHP Fatal Error / Warning 泄露在响应体中

**失败排查**：

| 错误现象 | 可能原因 |
|----------|---------|
| 500 错误 / Fatal | model() shim 未加载（重查 P0-12），或 use facade 替换遗漏 |
| 空 JSON `{}` | 数据库查询条件有误，检查 result_type=0 是否生效 |
| session 相关错误 | Session middleware 执行顺序（重查 P0-14 / P0-22） |

**完成后**：更新 INDEX.md P2-05 → [x]，并将 INDEX.md P2 行进度改为 `5/5 ✅`
