## P5-03 移除 All.php Controller Shim

**前提**：确认以下方法已无调用来源（shim 本身替代了这些方法，P3/P4 阶段的 success/error/assign/fetch 由 shim 处理，但此时 shim 已经是标准实现，可以保留为正式代码）。

**评估**：P5-03 实际上是将 All.php 中的 shim 方法**转正**而非删除：
- `success()` / `error()`：这是项目自己的跳转逻辑，应保留（作为正式方法，不是 shim）
- `assign()` / `fetch()`：同样保留，是合法的便捷方法

**实际操作**：
1. 移除方法上的 `@deprecated` 注释和 "Shim" 标记注释
2. 删除方法体中遗留的兼容代码（如有 TP5 specific 的判断逻辑）
3. 确保 `success()` / `error()` 中的 `jump` 模板路径正确

```bash
# 删除 All.php 中的 "TP8 过渡 Shim" 注释标记
sed -i '/===== TP8 Controller Shim/d' application/common/controller/All.php
sed -i '/===== \/TP8 Controller Shim/d' application/common/controller/All.php
```
