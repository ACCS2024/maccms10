## P3-03 Admin 控制器 input() 替换

**目标**：admin/controller/ 下 280 处 `input()` 调用

**先统计各文件**：
```bash
grep -rc "input(" application/admin/controller/ | sort -t: -k2 -rn | head -15
```

**替换规则**：
```php
// 获取整个 POST
input('post.') → $request->post()

// 获取单个 POST 字段
input('post.id')        → $request->post('id')
input('post.id/d', 0)   → (int)$request->post('id', 0)
input('post.name/s', '') → $request->post('name', '')

// 获取 GET 参数
input('get.page', 1)   → $request->get('page', 1)
input('get.')          → $request->get()

// 获取 param（GET+POST 合并）
input('param.id')      → $request->param('id')
input('param.')        → $request->param()

// 带过滤器的（/d /s /a 后缀是 TP5 特有）
input('post.id/d')     → (int)$request->post('id')
input('post.name/s')   → (string)$request->post('name')
input('post.ids/a')    → (array)$request->post('ids', [])
```

**sed 辅助替换简单模式**：
```bash
# 替换 input('post.') 整体形式
find application/admin/controller/ -name "*.php" -exec \
  sed -i "s/input('post\.')/\$request->post()/g" {} +

# input('get.') 形式
find application/admin/controller/ -name "*.php" -exec \
  sed -i "s/input('get\.')/\$request->get()/g" {} +
```

**注意**：`/d` `/s` `/a` 过滤器后缀需手动处理（或用 PHP 脚本），不能完全用 sed 自动化。

**确保 `$request` 可用**：
- admin 控制器方法中通过 `$this->request` 或方法注入 `\think\Request $request` 获取
- TP8 中 `$this->request` 在 `think\Controller` 基类中仍可用（如果 All.php 仍 extends Controller）

**验证**：
```bash
grep -rn "input(" application/admin/controller/ | grep -v "//\|#\|htmlspecialchars"
# 期望：0 行
```

**完成后**：更新 INDEX.md P3-03 → [x]

## P3-04 Addon.php 适配内化 Addons

**目标文件**：`application/admin/controller/Addon.php`

此文件负责后台插件管理（安装/卸载/启用/禁用），深度依赖 fastadmin-addons 的加载逻辑。

**分析改动点**：
```bash
grep -n "Hook\|Loader\|fastadmin\|addons" application/admin/controller/Addon.php | head -20
```

**主要改动**：

1. 调用 `\think\addons\*` 命名空间的改为 `\app\common\addons\*`（如有）

2. 调用 `Hook::listen/add` 的改为 `Event::listen/trigger`：
   ```php
   // 原
   Hook::listen('addon_install', $addon);
   // 改为
   \think\facade\Event::trigger('addon_install', $addon);
   ```

3. 调用 `Loader::addNamespace` 的删除（PSR-4 由 composer 处理）

4. AddonsLoader 的实例化调用路径更新（如有直接 new）

**验证**：安装/卸载任一 addon（如 adminloginbg）无报错。
