## P1-01 模型基类 Base.php 适配

**目标文件**：`application/common/model/Base.php`

**改动 1**：`initialize()` → `init()`

TP5 Model 的初始化钩子是 `protected function initialize()`，TP8 改为 `protected static function init()`：

```php
// TP5（删除）
protected function initialize() { ... }

// TP8（替换）
protected static function init() {
    // 原 initialize() 的内容
    parent::init();
}
```

**改动 2**：追加 `getError()` 兼容方法

TP8 Model 移除了 `getError()`，但项目中大量调用 `$model->getError()` / `$this->getError()`（grep 统计共 ~25 处分布在多个 model 文件）。在 Base.php 中封装一次，全部继承：

```php
protected string $_error = '';

public function getError(): string
{
    return $this->_error;
}

protected function setError(string $msg): void
{
    $this->_error = $msg;
}
```

**改动 3**：原有代码中所有 `$this->error = $msg` 形式（TP5 Model 的错误属性赋值）改为 `$this->setError($msg)`。

**grep 验证**：
```bash
grep -rn "getError\(\)" application/common/model/ | grep -v "Base.php"
# 确认所有调用点都是调用 Base.php 定义的方法（通过继承）
```

**完成后**：更新 INDEX.md P1-01 → [x]

## P1-05 修复 fetchSql(false) 调用

**背景**：TP5 的 `Db::name()->fetchSql(false)->select()` 中 `fetchSql(false)` 表示「不输出 SQL」（即正常执行）。TP8 中 `fetchSql()` 不接受参数，有参会报 Warning，且 `fetchSql()` 本身就是「输出 SQL 字符串」模式。

**正确改法**：`fetchSql(false)` 应直接删除（保留链式调用中的其他部分）。

**受影响位置**（已确认）：

1. `application/admin/controller/Safety.php:133`
   ```php
   // 原
   $list = Db::name($pre_tb)->field($field)->whereOr($where)->fetchSql(false)->select();
   // 改为
   $list = Db::name($pre_tb)->field($field)->whereOr($where)->select();
   ```

2. `application/admin/controller/Annex.php:211`
   ```php
   // 原
   $list = Db::name('Annex')->field('*')->where($where)->limit($limit_str)->fetchSql(false)->orderRaw('annex_time desc')->select();
   // 改为
   $list = Db::name('Annex')->field('*')->where($where)->limit($limit_str)->orderRaw('annex_time desc')->select();
   ```

3. `application/admin/controller/Annex.php:275`
   ```php
   // 原
   $list = Db::name($pre_tb)->field('*')->limit($limit_str)->fetchSql(false)->select();
   // 改为
   $list = Db::name($pre_tb)->field('*')->limit($limit_str)->select();
   ```

**验证**：
```bash
grep -rn "fetchSql(false)" application/ --include="*.php"
# 期望：0 行
```

**前置阅读**：[PITFALLS.md — F-A](PITFALLS.md#️-f-a-tp5-数组条件语法critical全站-526-处) + [F-B](PITFALLS.md#️-f-b-mac_search_wd_like-返回-tp5-格式critical12-处调用)

**影响严重性：CRITICAL** — common/model 被 api/controller 和 admin/controller 共同依赖，
这里的条件不修复会导致所有写操作和条件查询静默失效。

**修复方式参考 PITFALLS.md**，关键规则：
```php
// 最常见模式 — eq 直接值替换
$where['field'] = ['eq', $val];    → $where['field'] = $val;

// 比较操作

$where['status'] = ['neq', 0];     → where('status', '<>', 0)

// mac_search_wd_like() 的 OR LIKE — 见 PITFALLS.md F-B 修复策略
```

**mac_search_wd_like 修复策略（选一）：**
- **推荐**：将函数改为接受字段名参数，直接操作 query 对象（见 PITFALLS.md 选项 A）
- **快速**：每个调用方改为 closure（11 处，可一次性做完）
