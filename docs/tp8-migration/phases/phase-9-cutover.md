## 目标

在 TP8 + PHP 8.5 上完成**全量回归 + 安全复核 + 性能对照**,灰度后切生产,并备好回滚。

## 前置依赖

P0–P8 全部 `tp8-p<N>-done`。

本阶段不改业务,只做验收/发布工程:

| 项 | 说明 |
|---|---|
| 全量回归 | 冒烟矩阵 100% + 黄金 diff 全绿(前台/后台/API/install/插件) |

## 设计要点

- **数据库双向兼容**:迁移期未改 schema(只改代码)。确认 TP8 代码与现网 DB 完全兼容,使得"代码可回滚而数据不动"——这是安全回滚的前提。若某阶段确需 schema 变更,必须**向后兼容**并登记。
- **灰度顺序**:预发全栈回归 → 生产单机/小流量(8.4)→ 观察日志零新增 Fatal/Deprecated → 全量 8.4 → 切 8.5。
- **回滚触发线**:出现安全不变量破坏、支付/上传/登录核心流程故障、或错误率超阈 → 立即切回 TP5.0 镜像。
