# 00 · 迁移铁律 & 每轮工作循环

本文件是**强制规程**。任何一轮迁移工作(无论 AI 还是人执行)都必须遵守。违反 → 该轮作废、回滚。

---

## 一、八条铁律

1. **先分析,后动手。** 每一轮开工前,必须按 `templates/round-analysis-template.md` 产出《本轮分析》并存入 `progress/round-log/ROUND-XXX.md`。**没有分析报告,不许改一行代码。**
2. **切片要小。** 一轮只动一个"可独立验证"的切片(一个模块/一类 API/一组文件)。**禁止一轮跨多个阶段**。经验阈值:单轮改动 ≤ ~30 文件 / ≤ ~800 行,超了就拆。
3. **每轮必验证。** 改完必须在 **PHP 8.4 宿主**至少跑通 `php -l` + 启动 + 相关冒烟行(`verification/smoke-matrix.md`);触及安全脊柱的切片,必须跑 `verification/security-invariants.md` 对应项 + Docker 全栈复验。
4. **安全不变量不可破。** 见 `verification/security-invariants.md` 的 8 条。任一条在某轮被打破而未在同轮恢复 → 该轮**禁止合并**。
5. **功能等价优先于"写得漂亮"。** 迁移阶段不顺手重构业务逻辑、不改 SQL 语义、不改模板产出。想优化 → 记进 backlog,迁移完成后单独做。
6. **保留可回滚点。** 每轮一个聚焦提交(见 `02-conventions.md`);每阶段一个 tag。坏了能 `git revert` 单轮,不牵连。
7. **进度即时可见。** 每轮收工必须更新 `progress/PROGRESS.md`(勾选 + 当前轮指针 + 残留项)。看板与代码状态不一致 = 流程故障。
8. **残留必登记。** 本轮没做完/绕过/打了 TODO 的,全部写进该轮 round-log 的"残留"段和 PROGRESS 的"债务"区,不允许"悄悄留着"。

## 二、每轮工作循环(六步,缺一不可)

```
┌─ ① 分析 ANALYZE ──────────────────────────────────────────┐
│  · 复扫本切片现状(跑 grep/计数,别凭记忆)                  │
│  · 列出本轮要改的确切文件 + 调用点数                          │
│  · 写出 TP5→TP8 的映射(本切片)                              │
│  · 判定触碰了哪些安全不变量                                   │
│  · 定验证计划(宿主命令 + 是否需 Docker + 冒烟哪几行)         │
│  · 定回滚点                                                   │
│  → 产出 progress/round-log/ROUND-XXX.md(模板),提交此分析    │
└───────────────────────────────────────────────────────────┘
            ▼
② 设计 DESIGN   本切片的目标结构/接口落点(必要时贴新旧代码对照)
            ▼
③ 实现 IMPLEMENT  只改本切片;遵守"功能等价"
            ▼
④ 验证 VERIFY   宿主 PHP8.4:php -l → 启动 → 冒烟子集 → 安全不变量子集
                高风险切片:docker compose 全栈复验
            ▼
⑤ 记录 RECORD   回填 round-log(验证结果/残留/截图或输出)+ 勾 PROGRESS
            ▼
⑥ 提交 COMMIT   一轮一聚焦提交(见 02-conventions)
```

> **AI 执行强调**:第 ① 步的"复扫现状"必须**真的跑命令**(`grep -rEn ... | wc -l`、`php -l`、`docker compose ...`),把输出贴进 round-log,**不得用记忆或假设代替**。这是本方案对"AI 每轮都要分析"的硬性落地。

## 三、什么算"一轮做完了"(轮次 DoD)

- [ ] round-log 的《分析》在动手前已提交
- [ ] 本切片 `php -l` 全绿(PHP 8.4)
- [ ] 站点能启动(宿主 `php -S` 或 docker),相关页面/接口可访问无 500
- [ ] `verification/smoke-matrix.md` 中**本切片相关行**通过
- [ ] 触及的**安全不变量**逐条复验通过(若涉及)
- [ ] PROGRESS.md 已勾选、残留已登记
- [ ] 一个聚焦提交已落地(可单独 revert)

阶段级 DoD 见 `templates/phase-exit-criteria.md`。

## 四、风险分级(决定验证强度)

| 级别 | 切片类型 | 验证强度 |
|---|---|---|
| 🔴 高 | 安全行为/中间件(P3)、鉴权、支付、上传、插件加载 | 宿主 + **Docker 全栈** + 安全不变量全跑 + 人工复核 |
| 🟠 中 | 控制器/视图/taglib、数据层、配置 | 宿主启动 + 相关冒烟行 + 受影响不变量 |
| 🟢 低 | 纯命名空间/门面替换、helper 改名 | 宿主 `php -l` + 启动 + 1~2 冒烟行 |

> 拿不准级别 → 按高一级处理。
